GetActionURL

Descrição:

TODO: TRANSLATION Este metodo compoe um link para uma URL no sistema.
function GetActionURL($module='', $action='NONE', $item='', $args=null , $dispatch=null)

Parâmetros:

$module (string) Default ''
Nome do modulo que sera acessado
$action (string) Default 'NONE'
Nome do handler (.inc) a ser acessado
$item (string) Default ''
Parametro adicional que pode ser utilizado para passar dados para a nova pagina.
$args (string/array) Default null
Esse argumento pode ser utilizado para criar outras variaveis, alem das tres anteriores (que sao padrao do MIOLO). Quando for informado um array, o key sera o nome da variavel atraves do qual o conteudo podera ser acessado.
$dispatch (string) Default null
Indica qual arquivo devera ser utilizado ao inves daquele configurado no miolo.conf: $MIOLOCONF['options']['dispatch']

Retorna:

(string) o link para uma URL

Exemplo:

...
$module = 'reports';
$main   = 'main:list_person';

$handler = $MIOLO->GetActionURL($module, $main);
...
O exemplo acima retorna um link para o handler list_person.inc, no
modulo reports: http://nome_site/handler.php?module=reports&action=main:list_person

GetBusiness

Descrição:

TODO: TRANSLATION Como o MIOLO e capaz de abrigar modulos diferentes, era preciso adotar um esquema para evitar possiveis colisoes de nomes de classes. Teoricamente dois modulos poderiam definir uma classe, por exemplo, 'Guestbook' que, utilizada simultaneamente, causaria problemas.

O MIOLO espera que classes do tipo 'Business' tenham o seu nome composto de 'Business' + 'nome do module' + 'nome da classe'. Mas para evitar redundancias adotou-se o padrao de somente usar o nome basico da classe para definir o nome do arquivo, ja que o mesmo se encontra dentro da estrutura de diretorios do modulo em questao.
function GetBusiness($module,$name='main')

Parâmetros:

$module
Descrição do parametro $module
$name Default 'main'
Descrição do parametro $name

Exemplo:

/* in file: ../modules/foo/db/guestbook.class */
<?

class BusinessFooGuestbook extends Business
{
    function AddVisitor($data)
    {
          ...
?>


/* in file: ../modules/foo/handlers/register.inc */
<?
...
$guestbook = $MIOLO->GetBusiness( 'foo', 'guestbook' );
$data = $form->GetData();
result = $guestbook->AddVisitor( $data );
...
?>
Veja também: miolo/miolo.class#MIOLO::GetDatabase, miolo/ui/form.class#Form::GetData

_REQUEST

Descrição:

TODO: TRANSLATION No MIOLO, métodos que iniciam por "_" indicam métodos estáticos, mas neste caso foi utilizado para lembrar que sua funcionalidade é similar ao $_REQUEST do PHP. O metodo _REQUEST provê uma forma mais simples para se ter acesso às variáveis. Utilizando comandos PHP, vc. teria que utilzar $_REQUEST, $_GET, $_POST ou global ao passo que no MIOLO este método trás todas as informações, não importando de onde elas são provenientes. Caso você queira obter apenas o valor da variáveis provenientes de uma dessas opções, por exemplo GET, passe essa palavra como segundo parâmetro.
function _REQUEST($vars, $from='ALL')

Parâmetros:

$vars (string/array)
variáveis as quais se deseja obter o valor
$from (string) Default 'ALL'
de onde obter os dados. Pode ser 'GET', 'POST', além do padrão 'ALL' que retorna todos os dados.

Retorna:

(array) os valores das variáveis solicitadas
