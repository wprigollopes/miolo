# Internationalization (i18n)

[Back to Index](index.md)

## Overview

MIOLO uses [GNU gettext](https://www.gnu.org/software/gettext/) for internationalization. All translatable strings in PHP code are wrapped with the `_M()` function, and translations are stored in standard `.po`/`.mo` catalog files organized by language and module.

**Requirements:** PHP must be compiled with `--with-gettext`.

---

## Configuration

Set the locale path and language in `miolo.conf` (or in a module's `module.conf`):

```php
$MIOLOCONF['i18n']['locale']   = '/path/to/miolo/locale';
$MIOLOCONF['i18n']['language'] = 'pt_BR';
```

| Key | Description |
|-----|-------------|
| `i18n.locale` | Absolute path to the `locale/` directory containing translation catalogs |
| `i18n.language` | Language/locale code (e.g., `pt_BR`, `es`, `en_US`) |

---

## The `_M()` Function

Defined in `classes/support.php`, this is the primary translation function:

```php
function _M($msg, $dom = 'miolo', $p1 = null, $p2 = null, $p3 = null)
```

| Parameter | Description |
|-----------|-------------|
| `$msg` | The string to translate (used as the `msgid` in `.po` files) |
| `$dom` | The text domain -- corresponds to the `.po`/`.mo` filename (default: `'miolo'`) |
| `$p1`, `$p2`, `$p3` | Optional positional parameters, substituted for `@1`, `@2`, `@3` placeholders |

### Basic Usage

```php
// Simple translation (looks up in miolo.po)
echo _M('Save');

// Module-specific translation (looks up in admin.po)
echo _M('User Management', 'admin');
```

### Positional Parameters

Parameters allow word-order changes between languages:

```php
// English .po: msgid "@1 is a romance written by @2 in @3"
echo _M('@1 is a romance written by @2 in @3', 'mymodule',
        'Tom Sawyer', 'Mark Twain', '1890');
// Output: "Tom Sawyer is a romance written by Mark Twain in 1890"

// Portuguese .po: msgstr "@2 escreveu o romance @1 em @3"
// Output: "Mark Twain escreveu o romance Tom Sawyer em 1890"
```

The `@1`, `@2`, `@3` placeholders let translators reorder parameters to match the target language's grammar.

### How It Works Internally

1. Detects and converts ISO-8859-1 encoded strings to UTF-8 (legacy compatibility)
2. Switches the gettext text domain if `$dom` differs from the current domain
3. Calls PHP's native `_()` (gettext) function to look up the translation
4. Replaces `@1`, `@2`, `@3` placeholders with the provided parameters

## Translation Files

### File Types

| Extension | Description |
|-----------|-------------|
| `.po` | **Portable Object** -- human-editable source file containing `msgid`/`msgstr` pairs |
| `.mo` | **Machine Object** -- compiled binary catalog used by gettext at runtime |

### `.po` File Format

```po
# MIOLO i18n pt_BR file.
# Copyright (C) 2006-2011 Solis - Cooperativa de Soluções Livres
#
msgid ""
msgstr ""
"Project-Id-Version: 2.5\n"
"Content-Type: text/plain; charset=UTF-8\n"
"Language: pt_BR\n"

#: ../../classes/security/mperms.class.php:50
msgid "   Login   "
msgstr "   Login   "

msgid "   No   "
msgstr "Não"

msgid "   Yes   "
msgstr "   Sim   "

msgid " Cancel "
msgstr " Cancelar "
```

Each entry has:
- `msgid` -- the original string (must match exactly what `_M()` receives)
- `msgstr` -- the translated string
- Optional `#:` comments indicating source file references

### Compiling `.po` to `.mo`

Use the `msgfmt` command to compile the binary catalog:

```bash
msgfmt -f mymodule.po -o mymodule.mo
```

Or use the MIOLO CLI tool:

```bash
php bin/miolo.php translate extract   # extract translatable strings
php bin/miolo.php translate generate  # generate/compile catalogs
```

## Directory Structure

Translation files follow the standard Linux locale directory layout:

```
locale/
├── pt_BR/
│   └── LC_MESSAGES/
│       ├── miolo.po        # Core framework strings
│       ├── miolo.mo
│       ├── admin.po        # Admin module strings
│       ├── admin.mo
│       ├── common.po       # Common module strings
│       ├── common.mo
│       ├── example.po      # Example module strings
│       ├── example.mo
│       └── generator.po    # Generator module strings
├── es/
│   └── LC_MESSAGES/
│       ├── miolo.po        # Spanish translations
│       └── miolo.mo
└── README
```

Each module has its own `.po`/`.mo` file pair. The filename matches the domain parameter passed to `_M()`.

## Domain Switching

The `setDomainLocale()` function (in `classes/support.php`) handles switching between text domains:

```php
function setDomainLocale($domain)
{
    $lang      = $MIOLO->getConf('i18n.language');   // e.g., 'pt_BR'
    $charset   = $MIOLO->getConf('options.charset');  // e.g., 'UTF-8'
    $localedir = $MIOLO->getConf('i18n.locale');      // e.g., '/path/to/locale'

    setlocale(LC_ALL, "$lang.$charset", $lang, "$lang.utf8", "$lang.UTF-8");
    bind_textdomain_codeset($TEXTDOMAIN, $charset);
    bindtextdomain($TEXTDOMAIN, $localedir);
    textdomain($TEXTDOMAIN);
}
```

The `miolo` domain is initialized automatically at framework startup. When `_M()` is called with a different domain, it switches transparently.

## Adding a New Language

1. Create the directory structure:
   ```bash
   mkdir -p locale/fr/LC_MESSAGES
   ```

2. Copy an existing `.po` file as a template:
   ```bash
   cp locale/pt_BR/LC_MESSAGES/miolo.po locale/fr/LC_MESSAGES/miolo.po
   ```

3. Edit the `.po` file -- update the header and translate each `msgstr`:
   ```po
   msgid "Save"
   msgstr "Enregistrer"
   ```

4. Compile to `.mo`:
   ```bash
   msgfmt -f locale/fr/LC_MESSAGES/miolo.po -o locale/fr/LC_MESSAGES/miolo.mo
   ```

5. Configure the application:
   ```php
   $MIOLOCONF['i18n']['language'] = 'fr';
   ```

## Adding Translations for a Module

1. Create a `.po` file named after the module in the target language directory:
   ```bash
   touch locale/pt_BR/LC_MESSAGES/mymodule.po
   ```

2. Add the standard header and translation entries:
   ```po
   msgid ""
   msgstr ""
   "Content-Type: text/plain; charset=UTF-8\n"
   "Language: pt_BR\n"

   msgid "Student Name"
   msgstr "Nome do Estudante"

   msgid "Course"
   msgstr "Curso"
   ```

3. Use `_M()` with your module name as the domain:
   ```php
   $label = new MLabel(_M('Student Name', 'mymodule'));
   ```

4. Compile the `.po` file:
   ```bash
   msgfmt -f locale/pt_BR/LC_MESSAGES/mymodule.po -o locale/pt_BR/LC_MESSAGES/mymodule.mo
   ```
